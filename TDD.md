# FocusGuard AI — Technical Design Document

## 1. Overview

This document defines the architecture, data models, algorithms, and implementation design for FocusGuard AI — a Flutter mobile application for burnout prediction with AI-driven insights, voice control, and Firebase backend.

**Architecture:** MVVM (Model-View-ViewModel) + Service Layer

**Platform:** Android (primary), iOS (compatible)

**Language:** Dart / Flutter

---

## 2. Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| Framework | Flutter 3.x | Cross-platform UI (Material 3) |
| State Management | Provider | ChangeNotifier + reactive updates |
| Authentication | firebase_auth + google_sign_in | User login (email + Google) |
| Cloud Database | cloud_firestore | Per-user data storage |
| Firebase Core | firebase_core | Firebase initialization |
| Charts | fl_chart | Pie / line charts |
| AI | Groq API via http | Llama 3.3 70B for insights, recovery, chat, parsing |
| Voice | speech_to_text | On-device speech recognition |
| Fonts | Poppins | Typography |

## 3. Architecture

### Layer Diagram

```
┌─────────────────────────────────┐
│       Presentation Layer        │
│   (Screens, Widgets, Themes)    │
├─────────────────────────────────┤
│        ViewModel Layer          │
│  (ChangeNotifier + Provider)    │
├─────────────────────────────────┤
│         Service Layer           │
│  (Burnout, Prediction, AI,      │
│   Voice, Auth, Nutrition)       │
├─────────────────────────────────┤
│          Data Layer             │
│  (Models, Firestore repos)      │
├─────────────────────────────────┤
│        Firebase + Groq          │
│  (Auth, Firestore, AI API)      │
└─────────────────────────────────┘
```

### Responsibility Boundaries

- **Presentation:** Renders UI, captures input, observes ViewModel state
- **ViewModel:** Holds screen state, orchestrates services, exposes data via ChangeNotifier
- **Service:** Pure business logic — scoring, prediction, AI calls, voice, auth
- **Data:** Firestore repositories, models, serialization
- **External:** Firebase (Auth + Firestore) + Groq (AI inference)

## 6. Core Algorithms

### 6.1 Burnout Score

```dart
double calculate(UserInput input) {
  double sleepScore;
  if (input.sleepHours > 9) {
    sleepScore = ((input.sleepHours - 9) / 7) * 30; // oversleep penalty
  } else {
    final deficit = max(0, 8 - input.sleepHours);
    sleepScore = (deficit / 8) * 100;
  }
  final workScore = (min(input.workHours, 16) / 16) * 100;
  final moodScore = ((10 - input.mood) / 9) * 100;
  final screenScore = (min(input.screenTime, 16) / 16) * 100;
  final caffeineScore = (min(input.caffeine, 10) / 10) * 100;

  return (sleepScore * 0.30 + workScore * 0.25 + moodScore * 0.20
        + screenScore * 0.15 + caffeineScore * 0.10).clamp(0, 100);
}
```

Exercise bonus (-5) applied in ViewModel after calculation.

### 6.2 Nutrition Multiplier

```dart
// For 'nos' (eggs, chapati, bananas): quantity is count
// For 'grams' / 'ml': quantity is actual amount, divide by servingSize
final multiplier = food.unit == 'nos'
    ? log.quantity
    : log.quantity / food.servingSize;
totalProtein += food.protein * multiplier;
```

### 6.3 Prediction

```dart
tomorrow = currentScore * 0.85 + trend;
// trend = +8 (up), -5 (down), +3 (stable)
// 3-day: iterate, feeding predictions back as history
```

### 6.4 Simulation

```dart
improved = input.copyWith(
  sleepHours: max(current, 7.5),
  workHours: min(current, 8.0),
  mood: min(current + 2, 10),
  screenTime: min(current, 4.0),
  caffeine: min(current, 2),
);
improvedScore = calculate(improved);
```

## 7. AI Integration (Groq)

### 7.1 Why Groq
- **Free tier:** 30 RPM, no daily token quota limit during hackathon
- **Fast:** ~200ms response time (Llama 3.3 70B)
- **No credit card required**
- Replaced Gemini after quota issues

### 7.2 Service: `GeminiService` (kept name for continuity)

```dart
class GeminiService {
  final String _apiKey;
  static const _baseUrl = 'https://api.groq.com/openai/v1/chat/completions';
  static const _model = 'llama-3.3-70b-versatile';
  final List<DateTime> _callTimestamps = []; // rate limiter

  Future<String?> generateInsight({...});
  Future<List<Suggestion>?> generateRecoveryPlan({...});
  ChatSession startChat({...});
  Future<Map<String, dynamic>?> extractCheckinFromText(String text);
  Future<String?> generateFoodAdvice({...});
  Future<List<Map<String, dynamic>>?> parseFoodFromVoice(String text);
}
```

### 7.3 Rate Limiter

```dart
bool get _isRateLimited {
  _callTimestamps.removeWhere(
    (t) => DateTime.now().difference(t).inSeconds > 60);
  return _callTimestamps.length >= 28;
}
```

### 7.4 API Key Handling

- Passed via `--dart-define=GROQ_API_KEY=xxx` at build time
- Read via `const String.fromEnvironment('GROQ_API_KEY')`
- If empty → all AI features silently disabled, rule-based fallbacks used
- Stored in `.env` file (gitignored), loaded by `run.sh`

### 7.5 Fallback Strategy

Every AI call wraps in try/catch:
1. Gemini method returns null on any failure
2. ViewModel catches and uses rule-based service:
   - `generateInsight` fails → static `_insightFor(topCause)` map
   - `generateRecoveryPlan` fails → `RecommendationService.generate()`
   - `generateFoodAdvice` fails → hidden (rule-based rec shown instead)
   - `extractCheckinFromText` fails → "Try the sliders" snackbar
   - `chat` fails → "Sorry, I couldn't connect" bubble
3. No user-facing errors — graceful degradation

### 7.6 Firestore AI Cache

Per-user subcollection `users/{uid}/ai_cache/`:
- `insight_{date}` — one per check-in day
- `recovery_{date}`
- `food_{date}_{mealTime}`
- Prevents redundant API calls when re-opening a screen

### 8.2 Android Permissions

```xml
<uses-permission android:name="android.permission.RECORD_AUDIO"/>
<uses-permission android:name="android.permission.INTERNET"/>
```

### 8.3 MicButton Widget

- Tap to toggle listening
- Idle: yellow background + mic icon
- Listening: red background + stop icon
- `VoiceListeningOverlay` shows red pulsing dot + partial speech text

### 8.4 Append Mode

Voice results append to existing text:
```dart
onResult: (text) {
  final current = controller.text;
  controller.text = current.isEmpty ? text : '$current $text';
}
```

Users can tap mic multiple times to build up speech.

### 8.5 Three Use Cases

1. **Check-In NL sheet** → speech → Groq extracts → fills sliders
2. **Chat input** → speech → appends to message box → user sends
3. **Nutrition voice CTA** → speech → Groq parses food items → auto-adds to log

## 9. Firestore Schema

### Collection Structure

```
firestore-root/
├── users/{uid}/
│   ├── name, email, age, occupation
│   ├── check_ins/{date}/
│   │   └── sleep_hours, work_hours, mood, screen_time, caffeine, burnout_score, created_at
│   ├── nutrition_logs/{autoId}/
│   │   └── date, food_item_id, quantity, created_at
│   └── ai_cache/{docId}/
│       └── type, content, created_at
│
└── food_items/{id}/
    └── name, serving_size, protein, calories, carbs, fat,
        category, icon, unit, step_size
```

### Security Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid} {
      allow read, write: if request.auth.uid == uid;
      match /{subcol}/{doc} {
        allow read, write: if request.auth.uid == uid;
      }
    }
    match /food_items/{itemId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;  // allows re-seeding
    }
  }
}
```

### Food Re-seed Logic

If `food_items` missing the `unit` field (old data), the app auto-detects and re-seeds with updated data:

```dart
bool needsReseed = snapshot.docs.isEmpty;
if (!needsReseed) {
  final data = snapshot.docs.first.data() as Map<String, dynamic>?;
  if (data != null && !data.containsKey('unit')) needsReseed = true;
}
if (needsReseed) { deleteAll(); seedFresh(); }
```

## 14. Running the App

### Prerequisites

- Flutter SDK 3.x
- Android SDK + device in USB debugging mode
- Firebase project configured (`flutterfire configure`)
- Groq API key

### Environment Setup

Create `.env` file in project root:
```
GROQ_API_KEY=gsk_...
```

### Run Command

```bash
source .env && flutter run --dart-define=GROQ_API_KEY="$GROQ_API_KEY" -d <device_id>
```

Or use `bash run.sh -d <device_id>` (reads `.env` automatically).

### Without API Key

App runs with AI features silently disabled. All rule-based logic still works.

## 15. Performance Considerations

- All local calculations are O(1)
- Firestore queries indexed on `date` for fast history lookups
- Groq API calls are async and fire after result is rendered — no blocking
- AI insight/recovery cached per day to avoid duplicate API calls
- Provider rebuilds scoped using `Consumer`/`Selector` where possible
- Food grid is small (25 items) — no pagination needed
- Voice recognition runs on-device — no network latency
